# 設計書

## 1. アーキテクチャ

HTML、CSS、JavaScriptをすべて `src/index.html` に内包する1ファイルSPA。

外部CDN:

- Tailwind CSS
- Google Fonts
- Font Awesome

ローカル保存:

- 掲示板投稿を `localStorage` に保存

外部API通信:

- なし

## 2. 画面設計

### Welcome

目的: 企画の第一印象を決める扉ページ。

要素:

- タイトル
- 注意書き
- ガイドキャラ
- 診断開始ボタン

### Quiz

目的: 5問の選択で診断スコアを蓄積する。

要素:

- プログレスバー
- 質問カード
- 4択ボタン

### Loading

目的: 診断中の待ち時間を演出に変える。

要素:

- タコ行列
- メイン分析タコパン
- ランダム風メッセージ切替

### Result

目的: SNSで共有したくなる診断カードを表示する。

要素:

- 結果タイプ名
- パンツタイプ別の女の子ビジュアル
- 結果ギア画像
- サブキャッチ
- 解説文
- パラメータゲージ
- コピー、スクショ、SNS風シェア
- ツッコミ掲示板

## 3. データ設計

### questions

各質問は以下の形式:

```js
{
  text: "質問文",
  choices: [
    { text: "選択肢", scores: { tie: 3, seamless: 1 } }
  ]
}
```

### results

各結果は以下の形式:

```js
{
  name: "紐パンタイプ",
  icon: "🎀",
  gear: "tie",
  girl: "girl_tie",
  gradient: "...",
  sub: "サブキャッチ",
  params: {
    "メンタル安定度": 20,
    "警戒心": 15,
    "省エネ率": 35
  },
  desc: "解説文"
}
```

### localStorage

キー:

```text
jk_survival_board
```

値:

```json
[
  { "name": "現役JK", "text": "紐パン固結びしろは..." }
]
```

## 4. 状態管理

- `current`: 現在の質問番号
- `scores`: タイプ別スコア
- `currentResult`: 最終結果

状態はすべてクライアントメモリで保持し、掲示板のみlocalStorageに保存する。

## 4.5. タコパン演出設計

タコパンは、下着テーマを直接的に見せすぎず、結果前の待ち時間をギャグ演出にするための限定マスコット。

- Welcome: 出さない
- Quiz: 出さない
- Loading: 「鑑定中」ギミックの主役にする
- Result: 出さない
- Board: 出さない

注意: タコパンは常駐させない。存在理由が自然に通る「回答後から結果前」のみで使う。

## 5. デザイン設計

方向性:

- Y2K
- 平成レトロポップ
- ネオブルータリズム
- 太い黒枠
- 強いドロップシャドウ
- ネオンピンク、ミント、イエロー

注意:

- 下着テーマを直接的に見せすぎない
- パンツ画像は結果ギアとして扱う
- 女の子素材はタイプごとの世界観・表情・部屋演出として扱う
- キャラとタコの存在感を優先する
- 透過PNG素材は透明背景を保持し、銀色・グレーの不要な背景を合成しない
- 透過素材の背面は、白、黄色、ピンクなどのポップなUI面で受ける

## 6. セーフティ設計

- 「エロ期待厳禁」を扉ページに明示
- 性的な誘導や露骨な身体描写は入れない
- 未成年文脈ではギャグ、SNS、メンタル分析に寄せる
- 結果説明は偏見ギャグとして成立させる
